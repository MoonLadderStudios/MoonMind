# Implementation Plan: Live Log Tailing

**Branch**: `084-live-log-tailing` | **Date**: 2026-03-17 | **Spec**: [spec.md](file:///Users/nsticco/MoonMind/specs/084-live-log-tailing/spec.md)
**Input**: Feature specification from `specs/084-live-log-tailing/spec.md`

## Summary

Add a collapsible Live Output panel to the Mission Control task detail page that embeds the tmate `web_ro` URL as a read-only terminal view. The panel streams live agent terminal output on demand, disconnects when collapsed or backgrounded, and handles all session lifecycle states. The feature is UI-only (no new backend endpoints) and is gated behind a `logTailingEnabled` feature flag.

## Technical Context

**Language/Version**: JavaScript (vanilla), Python 3.11  
**Primary Dependencies**: Existing `dashboard.js` SPA, existing `task_runs.py` + `agent_queue.py` routers  
**Storage**: Existing `task_run_live_sessions` table (no schema changes)  
**Testing**: pytest (unit), browser-based manual verification  
**Target Platform**: Web browser (Mission Control dashboard)  
**Project Type**: Web application  
**Performance Goals**: Terminal view loads within 5 seconds of toggle; stream disconnects within 2 seconds of collapse  
**Constraints**: No new API endpoints; no new backend dependencies  
**Scale/Scope**: Single panel addition to task detail page

## Implementation Approach

### 1. Live Output Panel (dashboard.js)

Add a new `renderLiveOutputPanel(state)` function near the existing `renderQueueLiveSessionSection()`. The panel renders:

- A collapsible header with "Live Output" label and toggle button
- When opened and session is READY: an iframe pointed at the `web_ro` URL (existing field already fetched by `loadQueueRunDetail()`)
- When STARTING: a loading spinner
- When ENDED/REVOKED: "Session ended" message
- When DISABLED/ERROR or no session: "Live output is not available for this task"

The panel manages its own connection lifecycle:
- On expand: create iframe with `web_ro` src
- On collapse: remove iframe (clears the connection)
- On `visibilitychange` (tab hidden): remove iframe
- On `visibilitychange` (tab visible) + panel open: recreate iframe

### 2. Feature Flag (task_dashboard_view_model.py)

Add `logTailingEnabled` to the existing features configuration object exposed by the view model. Default to `true` when the live session system is available. The dashboard checks this flag before rendering the panel.

### 3. Existing Infrastructure Used

- `GET /api/queue/jobs/{id}/live-session` — already fetched by `loadQueueRunDetail()` in dashboard.js
- `web_ro` field — already available in the live session response (see `agent_queue_models.py` line 225)
- `task_run_live_sessions` table — no schema changes needed
- tmate web viewer — handles terminal rendering, scrollback, and rolling buffer natively
