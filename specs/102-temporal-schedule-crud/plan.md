# Implementation Plan: Temporal Schedule CRUD

**Branch**: `102-temporal-schedule-crud` | **Date**: 2026-03-23 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/102-temporal-schedule-crud/spec.md`

## Summary

Add Temporal Schedule lifecycle methods to `TemporalClientAdapter` so MoonMind can create, describe, update, pause/unpause, trigger, and delete Temporal Schedules. This includes mapping MoonMind's policy vocabulary (overlap, catchup, jitter) to Temporal SDK types and wrapping SDK exceptions in adapter-level errors.

## Design

### Policy Mapping Module (`schedule_mapping.py`)

A pure-function module that maps MoonMind policy inputs to Temporal SDK objects:

- `map_overlap_policy(mode: str) -> ScheduleOverlapPolicy`
- `map_catchup_window(mode: str) -> timedelta`
- `build_schedule_spec(cron: str, timezone: str, jitter_seconds: int) -> ScheduleSpec`
- `build_schedule_policy(overlap_mode: str, catchup_mode: str) -> SchedulePolicy`
- `build_schedule_state(enabled: bool, note: str) -> ScheduleState`
- `make_schedule_id(definition_id: UUID) -> str`
- `make_workflow_id_template(definition_id: UUID) -> str`

### Adapter Methods on `TemporalClientAdapter`

Each method gets the client via `self.get_client()`, constructs Temporal SDK objects using the mapping module, and wraps SDK exceptions.

```python
async def create_schedule(self, *, definition_id, cron, timezone, overlap_mode, catchup_mode, jitter_seconds, enabled, note, workflow_type, workflow_input, memo, search_attributes) -> ScheduleHandle
async def describe_schedule(self, *, definition_id) -> ScheduleDescription
async def update_schedule(self, *, definition_id, cron, timezone, overlap_mode, catchup_mode, jitter_seconds, enabled, note) -> None
async def pause_schedule(self, *, definition_id) -> None
async def unpause_schedule(self, *, definition_id) -> None
async def trigger_schedule(self, *, definition_id) -> None
async def delete_schedule(self, *, definition_id) -> None
```

### Exception Types (`schedule_errors.py`)

```python
class ScheduleAdapterError(Exception): ...
class ScheduleNotFoundError(ScheduleAdapterError): ...
class ScheduleAlreadyExistsError(ScheduleAdapterError): ...
class ScheduleOperationError(ScheduleAdapterError): ...
```

### Schedule ID Convention

- Schedule ID: `mm-schedule:{definition_uuid}`
- Workflow ID template: `mm:{definition_uuid}:{{.ScheduleTime}}`
