# Data Model: Auth Security Boundaries

## Credential Ref

- `profileId`: Provider profile identifier.
- `volumeRef`: Durable auth volume reference, never a raw listing or file payload.
- `volumeMountPath`: Explicit mount target path, treated as a ref/path only.
- `runtimeId`: Managed runtime identifier.

Validation rules:

- Credential refs may be serialized in workflow and browser-visible payloads.
- Credential file contents, raw directory listings, token values, private keys, cookies, and environment dumps must not be serialized.

## Sanitized Surface

Represents any operator-visible or durable output:

- OAuth session API responses.
- Provider profile API responses.
- Managed-session summaries and diagnostics.
- Workload stdout/stderr artifacts.
- Workload diagnostics artifacts.
- Workload result metadata.

Validation rules:

- Allowed values: status, timestamps, failure reasons, profile summaries, compact refs, declared output refs, and sanitized metadata.
- Forbidden values: raw credential files, token-like values, private key blocks, raw auth-volume listings, and full environment dumps.
- Secret-like strings are replaced with a redaction marker before persistence or response serialization.

## Workload Credential Declaration

Represents the policy gate for any future workload credential mount:

- `mountSource`: Named credential source or ref.
- `target`: Explicit container target path.
- `justification`: Non-empty human-readable reason.

Validation rules:

- Undeclared auth-like mounts fail validation.
- A declaration without justification fails validation.
- This story does not add a new credential-requiring workload profile; it documents and tests the fail-closed behavior for current workload profiles.

## Boundary Verification Evidence

Evidence generated by tests and runtime validation:

- `surface`: Boundary under test.
- `inputFixture`: Sanitized description of secret-like fixture data.
- `result`: Pass/fail with sanitized reason.
- `coverageIds`: Source design coverage IDs proven by the check.

Validation rules:

- Evidence must not include the secret fixture values themselves.
- Evidence must map to the MM-335 source coverage IDs.
